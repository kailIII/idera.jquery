Métodos

idera()

idera(opciones:Object|string)

Este método se llama una sola vez.
Inicializa un elemento html para poder ser transformado en algún widget de idera a través de los widgets.

Todos los demás métodos se usan después de usar éste y pueden concatenarse $('#autocomplete').idera().autocompleteDeCapas()

opciones
(objeto simple, opcional): Si opciones no es una string, se interpreta como pares de key/valor que configuran opciones extra para el comportamiento del plugin:
id_nodo
(string). El identificador de uno de los nodos WMS de IDERA. La lista de identificadores la podés encontrar en Datos de IDERA.
debug
(boolean). Si es true, el plugin es más verborrágico en la consola. El valor predeterminado es false.
opciones
(string, opcional): Si opciones es una string, se toma como el identificador de uno de los nodos WMS de IDERA. Es una manera rápida de hacer lo mismo que pasando la propiedad id_nodo cuando opciones se pasa como objeto.

mapa(capa:string)

Con .mapa() para mostrar una tabla con todas las Capas WMS que conforman la IDERA.

capa
(string): El nombre de la capa que se quiere superponer sobre el mapa

El servicio WMS tiene que estar configurado para poder generar las imágenes con el sistema de referencia EPSG:3857 que es el código que define la proyección de OpenStreetMap, Google Maps, Bing Maps, ESRI WEB Maps y demás mapas web populares.

tablaDeCapasWMS()

Luego de llamar a idera(), puede usarse .tablaDeCapasWMS() para mostrar una tabla con las capas WMS de uno de los servicios WMS de la IDERA.

prosa()

Luego de llamar a idera(), puede usarse .prosa() para mostrar un texto descriptivo con la información declarado por un servicios WMS de la IDERA.

eventos

  • idera.afterWMSCapabilitiesParsed
  • idera.afterWMSLayerParsed

idera.afterWMSCapabilitiesParsed

Este evento se dispara apenas se termina de parsear un documento capabilities completo.

La función de callback que se le pasa a .on() recibe el objeto parseado como par de key/values en sus segundo parámetro.

capabilities
(objeto simple): Las propiedades del servicio WMS en un objeto simple.
idera.afterWMSLayerParsed

Se dispara apenas se termina de parsear una capa WMS a partir del elemento XML que la representa en un documento capabilities que está siendo parseado.

La función de callback que se le pasa a .on() recibe el objeto parseado como par de key/values en sus segundo parámetro.

capa
(objeto simple): Las propiedades de la capa WMS recién analizada en un objeto simple.
